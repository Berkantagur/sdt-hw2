# Software Development Tools – Homework 2
This repository created for Software Development Course's Homework.

> **Subject:** GIT  
> **repo** = repository  

Using a GIT server (GitHub/GitLab) and a GIT client, perform the following steps **both from the IDE and from the terminal**.  
You can use any IDE, e.g. **NetBeans**, **MS Visual Studio**, **VS Code**, **IntelliJ**, etc.

---

## Task Overview

GIT – project on a local machine with a remote repository (GitHub/GitLab).

### Main Steps (a–p)

- [ ] **a)** Create a **remote repository** (on GitHub/GitLab, …).  
- [ ] **b)** **Clone** the empty repository (`git clone`).  
- [ ] **c)** In the local repo folder, create an **empty project** using the chosen language.  
- [ ] **d)** Commit the whole project to the repo  
      `git commit -m "Initial project"`  
- [ ] **e)** Add simple code to the Java/C++/C# project  
      (e.g. create an array/table and print its elements).  
- [ ] **f)** Commit the changes.  
- [ ] **g)** Add more code (e.g. **initialize the table with random values**).  
- [ ] **h)** Commit the changes.  
- [ ] **i)** Add more code (e.g. **sort the table elements**).  
- [ ] **j)** Commit the changes.  
- [ ] **k)** Look at the code history using  
      `git log`.  
- [ ] **l)** Look at code annotations using  
      `git blame`.  
- [ ] **m)** Try to check out different revisions using  
      `git checkout <commit-hash>`.  
- [ ] **n)** Add some changes to the code, **do not commit**.  
- [ ] **o)** Try to revert the last changes (e.g. using `git restore` or  
      `git revert` for the last commit).  
- [ ] **p)** Push the project to the remote repository  
      `git push`.

---

### Repository & Branch Management (r–y)

- [ ] **r)** Delete the local project and local repository.  
- [ ] **s)** Clone the project again from the remote repository (`git clone`).  
- [ ] **t)** Create a **tag/release** and try to switch between this tag and the
      `master/main` branch (`git branch`, `git tag`, `git checkout <tag>`).  
- [ ] **u)** Create a **new branch** from the `master/main` branch.  
- [ ] **w)** Switch to the new branch (`git checkout <branch>` / `git switch <branch>`).  
- [ ] **x)** Improve your code in this branch (e.g. **change the sorting algorithm**).  
- [ ] **y)** Merge the new branch into the `master/main` branch  
      (`git merge <branch>`).

---

### Collaboration & Conflicts (z–z3)

- [ ] **z)** Give the URL of the repository to one of your friends  
      (first, allow him/her access to the repo).  
- [ ] **z1)** Try to **produce a merge conflict**.  
- [ ] **z2)** **Solve** the conflict and push the solution to the remote repository.  
- [ ] **z3)** Send the repository URL and a **short report** to the teacher  
      (do not forget to give the teacher access to the repository if needed).
